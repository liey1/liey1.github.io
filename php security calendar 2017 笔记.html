这个笔记也只是过一遍大概的，这也不是全的，有一些跳过了，比如 XXE 的 :)

day1

var_dump(1==”1a”);

day11

看注释。，漏洞就是 任意文件写入。

这里要绕过两个，就是 loadData 里的。第一个是不能 O: 开头，于是我们可以加个 Array 在 Object 对面外面，大概就是：

Array(new Test())

第二个就是不能有 O:数字，于是我们可以 O:+数字，这个 + 表示正数吧。。但是这个操作在 7.2 就不顶用了。。

这里要提一下，有个 Tips 是把 Object 的属性调大，可以绕过 unserialize 调用的 __wakeup 魔术方法，就是这样:

unserialize(‘O:4:”Test”:100:{s:3:”ttt”;s:1:”b”;}’);

但是，我在 php5.6 就已经不顶用了，我发现了个线上执行的，是 php 5.3 ，可以用：

day13

我佛了。。DQL (Doctrine Query Language)。。我把它注释了，本质上和 SQL 一样。

这里的问题在 sanitizeInput。addslashes 会把 ' 变成 \'。但是这里还有个截断，所以我们的 payload 可以这样：

user=aaaaaaaaaaaaaaaaaaa’&passwd= or 1=1%23

输出：

select * from test where user = ‘aaaaaaaaaaaaaaaaaaa\’ AND password = ‘ or 1=1#’

感悟一下吧。。。
